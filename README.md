## 課題

駅と路線のグラフから、同じ駅を2度通らずに総距離が最長になる経路を探します。

## 動作環境

- Python 3.7
- Windows 11 + Python 3.12 で動作確認しています。

## 実行方法

```
python longest_path.py < input.txt
```

経路の駅IDがCRLF区切りで標準出力に出ます。
制限時間はデフォルト10秒に設定しています。
総距離などの情報は標準エラーとして出しているので、出力をリダイレクトしても混ざりません。

テストの実行:

```
python -m unittest
```

## 入力の前提

- 路線は双方向(無向グラフ)として扱っています
- 同じ駅間に複数の路線がある入力も想定し、経路には重みが最大の1本を使います
- 始点と終点は同じ駅にできるため、閉路も候補に含めます
- 形式が正しくない行は読み飛ばします

## 解法

最長単純路問題はNP困難なので、2段構成にしています。

1. 分枝限定法(DFS+上界での枝刈り)で厳密解を試す。制限時間の6割以内に探索し切れたら最適解として終了する。
2. 終わらない規模なら、その時点のベストを初期解にして「経路の端を切り落として貪欲に伸ばし直す」局所探索に切り替え、残り時間いっぱい改善する。

ルールは始点と終点は同じでもよいということなので閉路も探索します。平行辺の場合での往復 a→b→a や自己ループも、閉路の特殊ケースとして候補に入れています。どちらのフェーズで答えを出したかは標準エラーのexactまたはapproxで分かります。

## テストケース

- 問題文の入力例 → 閉路 1→2→3→4→1(総距離 17.05)が返ります
- 5000駅・7500路線のランダムグラフ → 8秒で約2400駅を通る経路が出ました
